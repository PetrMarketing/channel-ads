## 1. Начало работы

### Как подключить канал

1. Откройте бота в MAX
2. Отправьте `/start`
3. Добавьте бота администратором в канал: Канал → Настройки → Администраторы → добавить бота
4. Бот обнаружит канал автоматически (может занять до 5 минут)
5. Войдите в веб-панель через кнопку «Личный кабинет»

### Как войти в личный кабинет
- Через бота: нажмите кнопку «Личный кабинет» в боте

### FAQ: Начало работы

**Q: Бот не видит мой канал после добавления**
A: Убедитесь, что бот добавлен именно как администратор (не просто участник). Подождите до 5 минут — права выдаются с задержкой.

**Q: Как подключить несколько каналов?**
A: Добавьте бота администратором в каждый канал. Все каналы появятся в личном кабинете.

## 2. Обзор и управление каналами

### Описание
Главная страница с обзором текущего канала: статистика, настройки, быстрые действия.

### Возможности
- Переключение между каналами
- Статистика: визиты, подписчики, лиды, запланированные посты
- Настройки канала: Яндекс Метрика, VK Пиксель, ссылка для вступления
- Удаление/переподключение каналов

### Настройки канала
| Поле | Описание | Пример |
|------|----------|--------|
| Яндекс Метрика ID | Номер счётчика из metrika.yandex.ru | `12345678` |
| VK Pixel ID | ID пикселя из ads.vk.com | `3751584` |
| Ссылка для вступления | Invite-ссылка канала | `https://t.me/+ABC123` |
| YM OAuth Token | Токен для загрузки офлайн-конверсий | — |

### FAQ

**Q: Как обновить invite-ссылку канала?**
A: Перейдите в настройки канала на странице «Обзор» и вставьте новую ссылку в поле «Ссылка для вступления». Нажмите «Сохранить».

**Q: Канал показывает статус «Деактивирован»**
A: Бот был удалён из администраторов канала. Добавьте бота обратно: Канал → Настройки → Администраторы.

## 3. Трекинг-ссылки

### Описание
Создание уникальных ссылок для отслеживания трафика из разных источников.

### Типы ссылок
- **Лендинг** — открывает страницу подписки с отслеживанием через Яндекс Метрику и VK Пиксель. Пользователь переходит → подписывается → система фиксирует конверсию.
- **Прямая** — мгновенный редирект в канал. Без лендинга, только внутренняя статистика кликов и подписок.

### Как создать ссылку
1. Перейдите в раздел «Маркетинг → Ссылки»
2. Нажмите «+ Создать ссылку»
3. Введите название (для себя)
4. Выберите тип: Лендинг или Прямая
5. Заполните UTM-метки (необязательно)
6. Нажмите «Сохранить»

### UTM-метки
| Параметр | Описание | Пример |
|----------|----------|--------|
| utm_source | Источник трафика | `telegram`, `vk`, `google` |
| utm_medium | Тип трафика | `cpc`, `post`, `banner` |
| utm_campaign | Название кампании | `spring_sale` |
| utm_content | Контент/креатив | `banner_1` |
| utm_term | Ключевое слово | `keyword` |

### Настройка пикселей на ссылке
Для лендинг-ссылок доступна кнопка «Пиксели»:
- **Яндекс Метрика**: ID счётчика + название цели (по умолчанию `subscribe_channel`)
- **VK Пиксель**: ID пикселя + название цели

Если на ссылке не указан счётчик — используется счётчик из настроек канала.

### FAQ

**Q: Чем отличается лендинг от прямой ссылки?**
A: Лендинг показывает промежуточную страницу с кнопкой «Перейти в канал» и отслеживает подписку. Прямая ссылка мгновенно перенаправляет в канал — подписка отслеживается только через бота.

**Q: Как работает отслеживание подписки?**
A: Пользователь переходит по лендинг-ссылке → бот проверяет, подписался ли пользователь → при подписке фиксируется конверсия и отправляется цель в Метрику/VK.

**Q: Можно ли приостановить ссылку?**
A: Да. Нажмите кнопку «Пауза» рядом со ссылкой. При переходе по приостановленной ссылке пользователь увидит ошибку.

## 4. Закрепы и лид-магниты

### Закрепы (Pinned Posts)
Создание и публикация закреплённых сообщений в канале.

**Возможности:**
- Создание с текстом, форматированием, вложениями (фото, видео, документ, аудио)
- Инлайн-кнопки (ссылки, лид-магниты, комментарии)
- Публикация в канал одним нажатием
- Прикрепление лид-магнита к закрепу
- Отслеживание лидов с конкретного закрепа

### Лид-магниты
Бесплатный контент в обмен на подписку / данные.

**Как создать:**
1. В разделе «Закрепы» перейдите на вкладку «Лид-магниты»
2. Нажмите «+ Создать»
3. Заполните: название, текст сообщения, вложение
4. Настройте кнопки (необязательно)
5. Включите «Только подписчикам» если нужно проверять подписку

**Типы кнопок:**
- URL-ссылка
- Лид-магнит (переход к другому лид-магниту)
- Комментарии (открывает мини-апп комментариев)

### FAQ

**Q: Как прикрепить лид-магнит к закрепу?**
A: При создании закрепа выберите тип кнопки «Лид-магнит» и укажите нужный лид-магнит из списка.

**Q: Что значит «Только подписчикам»?**
A: Перед выдачей лид-магнита бот проверит, подписан ли пользователь на канал. Если нет — предложит подписаться.

**Q: Поддерживаются ли вложения (фото, видео)?**
A: Да. При создании закрепа или лид-магнита можно прикрепить файл: фото, видео, документ или аудио.

## 5. Рассылки

### Описание
Массовая отправка сообщений подписчикам/лидам канала.

### Как создать рассылку
1. Перейдите в «Маркетинг → Рассылки»
2. Нажмите «+ Создать рассылку»
3. Введите название и текст сообщения
4. Настройте таргетинг (кому отправлять)
5. Добавьте инлайн-кнопки (необязательно)
6. Прикрепите файл (необязательно)
7. Настройте время отправки

### Таргетинг (фильтры получателей)
- **Все лиды** — все пользователи, взаимодействовавшие с каналом
- **По лид-магниту** — только получатели конкретного лид-магнита
- **По дате регистрации** — до/после определённой даты
- **По участию в розыгрыше** — участники конкретного розыгрыша

### Типы задержки
| Тип | Описание | Пример |
|-----|----------|--------|
| Через N времени | Относительная задержка | Через 2 часа |
| В день/время | Конкретный день недели + время | Понедельник в 10:00 |
| Точная дата | Конкретная дата и время | 15.04.2026 в 14:30 |

### FAQ

**Q: Можно ли протестировать рассылку перед отправкой?**
A: Да. Используйте кнопку «Тест» — сообщение придёт только вам (владельцу канала).

**Q: Сколько сообщений можно отправить?**
A: Ограничений нет, но Telegram и MAX имеют свои лимиты на частоту отправки. Система автоматически соблюдает лимиты с retry при ошибке 429.

**Q: Как узнать сколько людей получат рассылку?**
A: При настройке фильтров система автоматически подсчитывает количество получателей.

## 6. Воронки

### Описание
Автоматические серии сообщений, отправляемые пользователю по расписанию после получения лид-магнита.

### Как работает
1. Пользователь получает лид-магнит
2. Запускается воронка, привязанная к этому лид-магниту
3. Через заданные интервалы автоматически отправляются следующие шаги

### Настройка
1. Перейдите в «Маркетинг → Воронки»
2. Создайте шаги с текстом и задержкой
3. Привяжите воронку к лид-магниту
4. Активируйте

### FAQ

**Q: Можно ли перезапустить воронку для пользователя?**
A: Да. При повторном получении лид-магнита воронка перезапускается автоматически.

**Q: Что если пользователь отписался?**
A: Следующие шаги воронки не будут отправлены пользователям, которые отписались от канала.

## 7. Публикации (Контент)

### Описание
Планирование и публикация постов в канале по расписанию.

### Возможности
- Создание постов с текстом и вложениями
- Планирование на конкретную дату и время
- Публикация в Telegram и MAX
- Статусы: Черновик → Запланирован → Опубликован

### FAQ

**Q: Пост не опубликовался в запланированное время**
A: Проверьте, что бот имеет права на публикацию сообщений в канале. Также убедитесь, что подписка канала активна.

**Q: Можно ли редактировать опубликованный пост?**
A: Нет. Редактирование доступно только для черновиков и запланированных постов.

## 8. Розыгрыши

### Описание
Создание конкурсов и розыгрышей с автоматическим определением победителей.

### Как создать розыгрыш
1. Перейдите в «Контент → Розыгрыши»
2. Нажмите «+ Создать»
3. Заполните: название, описание, призы
4. Укажите дату окончания и количество победителей
5. Включите требование подписки (при необходимости)
6. Опубликуйте в канал

### Настройки
- **Название и описание** — отображаются в посте
- **Призы** — список призов с описанием
- **Дата окончания** — когда розыгрыш завершается
- **Количество победителей** — сколько человек выигрывают
- **Подписка обязательна** — участники должны быть подписаны на канал
- **ERID** — рекламный идентификатор для маркировки
- **Юридическая информация** — обязательная информация по закону о рекламе

### FAQ

**Q: Как определяются победители?**
A: Нажмите кнопку «Провести розыгрыш» — система случайным образом выберет победителей из числа участников.

**Q: Участник отписался от канала после участия**
A: Если включено требование подписки, отписавшиеся пользователи исключаются из розыгрыша.

## 9. Аналитика

### Описание
Статистика канала: подписчики, просмотры, вовлечённость.

### Метрики
- Подписчики (рост/убыль)
- Просмотры постов
- Engagement Rate (ER)
- Реакции
- Комментарии

### Периоды
- 7 дней
- 14 дней
- 30 дней
- 90 дней

### FAQ

**Q: Данные аналитики обновляются не сразу**
A: Сбор аналитики происходит каждые 6 часов. После первого подключения канала данные появятся в течение 6 часов.

**Q: Почему нет данных за прошлые периоды?**
A: Аналитика собирается с момента подключения канала. Исторические данные недоступны.

## 10. Комментарии

### Описание
Система комментариев для постов канала через мини-приложение.

### Как работает
1. При публикации поста добавьте кнопку типа «Комментарии»
2. Подписчики нажимают кнопку → открывается мини-приложение с комментариями
3. Комментарии отображаются в реальном времени

### Модерация
В разделе «Контент → Комментарии» вы можете:
- Просматривать все комментарии
- Удалять нежелательные комментарии
- Настраивать внешний вид виджета (цвет, заголовок)

### FAQ

**Q: Комментарии не отображаются**
A: Убедитесь, что кнопка типа «Комментарии» добавлена к посту и пост опубликован через систему.

## 11. Платные чаты

### Описание
Монетизация через платный доступ к приватным чатам/каналам.

### Настройка
1. Перейдите в «Монетизация → Платные чаты»
2. Настройте платёжную систему (Yoomoney, Robokassa, Tinkoff, Prodamus, GetCourse)
3. Создайте тарифные планы (цена, срок)
4. Привяжите приватный чат
5. Опубликуйте предложение в канале

### Вкладки раздела
| Вкладка | Описание |
|---------|----------|
| Чаты | Управление привязанными приватными чатами |
| Тарифы | Настройка планов подписки (цена, срок, описание) |
| Участники | Список подписчиков с датами доступа |
| Оплата | Настройка платёжных систем (ключи, токены) |
| Уведомления | Шаблоны уведомлений (приветствие, продление, истечение) |
| Публикация | Создание рекламного поста с предложением подписки |

### Платёжные системы
| Система | Что нужно |
|---------|-----------|
| Yoomoney | ID кошелька + Secret Key |
| Robokassa | Login + Password1 + Password2 |
| Tinkoff | Terminal Key + Password |
| Prodamus | API Key + Secret |
| GetCourse | Account + Secret Key |

### FAQ

**Q: Пользователь оплатил, но не получил доступ**
A: Проверьте правильность настроек платёжной системы (ключи, webhook URL). Также убедитесь, что бот является администратором приватного чата с правом приглашения пользователей.

**Q: Как работает продление?**
A: За N дней до окончания подписки пользователю автоматически отправляется уведомление о продлении (настраивается во вкладке «Уведомления»).

**Q: Как вручную добавить участника?**
A: Во вкладке «Участники» нажмите «+ Добавить» и укажите Telegram/MAX ID пользователя.

## 12. Услуги и запись

### Описание
Система онлайн-записи на услуги с календарём, специалистами и филиалами.

### Вкладки
| Вкладка | Описание |
|---------|----------|
| Филиалы | Адреса, телефоны, рабочие часы |
| Специалисты | Профили сотрудников, фото, расписание |
| Услуги | Каталог услуг с ценами и длительностью |
| Записи | Управление бронированиями |
| Клиенты | База клиентов |
| Уведомления | Шаблоны подтверждений и напоминаний |
| Оформление | Брендирование виджета записи |

### Как настроить
1. Создайте филиал (название, адрес, рабочие часы)
2. Добавьте специалистов (имя, должность, фото, расписание)
3. Создайте услуги (название, цена, длительность)
4. Привяжите услуги к специалистам
5. Опубликуйте виджет записи через бота

### Рабочие часы
Настраиваются для каждого филиала и специалиста отдельно. Формат: начало и конец рабочего дня для каждого дня недели.

### FAQ

**Q: Клиент не видит свободных слотов**
A: Проверьте, что рабочие часы специалиста настроены, услуга привязана к специалисту, и выбранная дата является рабочим днём.

**Q: Как ограничить количество записей в день?**
A: В настройках специалиста укажите «Максимум записей в день».

**Q: Как отменить запись?**
A: Во вкладке «Записи» найдите бронирование и нажмите «Отменить».

## 13. Сотрудники

### Роли
| Роль | Доступ |
|------|--------|

### Как пригласить сотрудника

2. Выберите роль
3. Нажмите «Создать приглашение»
4. Отправьте сгенерированную ссылку сотруднику
5. Сотрудник переходит по ссылке и получает доступ

### FAQ

**Q: Сотрудник не видит канал после приглашения**
A: Убедитесь, что сотрудник перешёл по ссылке-приглашению и авторизовался.

**Q: Как удалить сотрудника?**
A: В разделе «Сотрудники» нажмите «Удалить» рядом с нужным сотрудником.

## 14. Подписка и оплата

### Описание
Управление подпиской на сервис PK Marketing.

### Тарифы
Цены устанавливаются администратором в панели /admin/tariffs.

### Скидки
- Скидка 10% за каждый дополнительный канал
- Максимальная скидка: 90%

### Как оплатить
1. Перейдите в раздел «Подписка»
2. Выберите каналы и количество пользователей
3. Выберите срок подписки
4. Введите email для получения чека
5. Нажмите «Оплатить»
6. Завершите оплату на странице Тинькофф

### FAQ

**Q: Оплата не прошла**
A: Проверьте, что введён корректный email. Попробуйте другой способ оплаты. Если ошибка повторяется — обратитесь в поддержку.

**Q: Как продлить подписку?**
A: Оплатите новый период — он будет добавлен к текущему сроку.

**Q: Могу ли я оплатить за несколько каналов сразу?**
A: Да. На странице «Подписка» отметьте нужные каналы и оплатите одним платежом.

## 16. Пиксели и аналитика

### Яндекс Метрика
Для отслеживания конверсий (подписок) через Яндекс Метрику:

1. Создайте счётчик на https://metrika.yandex.ru
2. Скопируйте ID счётчика (только цифры)
3. Укажите ID в настройках канала или на конкретной ссылке
4. При подписке через лендинг-ссылку автоматически отправляется цель

**Офлайн-конверсии:**
Система также поддерживает загрузку офлайн-конверсий в Метрику через OAuth-токен.

### VK Пиксель
Для ретаргетинга через VK Рекламу:

1. Создайте пиксель в https://ads.vk.com
2. Скопируйте ID пикселя (числовой)
3. Укажите ID в настройках канала или на конкретной ссылке
4. При посещении лендинга — `pageView`, при подписке — `reachGoal`

### FAQ

**Q: Конверсии не отображаются в Метрике**
A: Убедитесь, что ID счётчика указан правильно (только цифры). Проверьте, что цель с указанным именем создана в настройках счётчика Метрики.

**Q: Какое название цели по умолчанию?**
A: `subscribe_channel`. Можно изменить в настройках ссылки (кнопка «Пиксели»).

## 18. Ошибки и решения

### Общие ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| 401 Unauthorized | Истёк токен авторизации | Перезайдите через бота |
| 403 Forbidden | Нет доступа к каналу | Убедитесь, что вы владелец или сотрудник канала |
| 404 Not Found | Ресурс не найден | Проверьте URL, ссылка могла быть удалена |
| 410 Gone | Ссылка приостановлена | Активируйте ссылку (снимите паузу) |
| 500 Internal Server Error | Серверная ошибка | Попробуйте повторить. Если ошибка повторяется — обратитесь в поддержку |
| 502 Bad Gateway | Ошибка платёжной системы | Проверьте настройки платёжной системы (ключи, токены) |

### Ошибки при публикации

| Ошибка | Причина | Решение |
|--------|---------|---------|
| «Бот не является администратором» | Бот удалён из админов канала | Добавьте бота обратно в администраторы |
| «Текст сообщения не может быть пустым» | Пустое сообщение | Заполните текст сообщения |
| Публикация проходит, но без файла | Файл не сохранён или повреждён | Перезагрузите файл и повторите публикацию |
| Ошибка 429 (Too Many Requests) | Превышен лимит API | Подождите 1-2 минуты, система автоматически повторит |

### Ошибки платежей

| Ошибка | Причина | Решение |
|--------|---------|---------|
| «Укажите email для отправки чека» | Не указан email | Заполните поле email на странице оплаты |
| Ошибка 309 (Тинькофф) | Некорректный чек | Проверьте настройки терминала Тинькофф |
| «Payment init failed» | Неверные ключи терминала | Проверьте TINKOFF_TERMINAL_KEY и TINKOFF_PASSWORD в .env |
| Оплата прошла, но подписка не активирована | Webhook не дошёл | Проверьте NotificationURL в настройках Тинькофф и APP_URL в .env |

### Ошибки ботов

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Бот не отвечает | Контейнер упал или бот не запущен | `docker-compose up -d --build` |
| Бот не видит канал | Нет прав администратора | Добавьте бота администратором канала |
| Ошибка создания invite-ссылки | Бот не имеет прав на приглашение | Дайте боту право «Приглашение пользователей» |
| MiniApp не открывается | Неверный APP_URL или DNS | Проверьте APP_URL в .env и DNS-записи |

### Ошибки деплоя

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Container unhealthy | Приложение не стартует | `docker logs channel-ads --tail 50` — смотрите traceback |
| Port already in use | Занят порт 8010 | `docker-compose down && docker-compose up -d` |
| Database connection error | PostgreSQL не доступен | Проверьте `docker-compose ps` — контейнер `db` должен быть healthy |
| Миграция не прошла | Ошибка SQL | Проверьте логи: `docker logs channel-ads 2>&1 \| grep migration` |
| Frontend не обновился | Кэш Docker | `docker-compose build --no-cache && docker-compose up -d` |
| SSL ошибка | Нет сертификата | `certbot --nginx -d domain.ru` |

### Как получить логи для диагностики
```bash
# Последние 50 строк логов
docker logs channel-ads --tail 50

# Логи в реальном времени
docker logs channel-ads -f

# Поиск ошибок
docker logs channel-ads 2>&1 | grep -A5 "Traceback\|Error\|500"

# Перезапуск
docker-compose restart app

# Полная пересборка
docker-compose up -d --build
```
